# Support

Glacier Engine is maintained by volunteers and currently offers community
support through GitHub issues.

## Ask a question

Before opening an issue:

1. Check the [Quickstart](docs/QUICKSTART.md) and
   [Glossary](docs/GLOSSARY.md).
2. Search existing issues for the command or error name.
3. Reduce the problem to the smallest reproducible command.

Include the Glacier commit, Zig version, operating system, architecture, build
flags, and complete error text. For benchmark questions, attach the machine
envelope and artifact rather than only a throughput number.

## Where each request belongs

- Bug or reproducible failure: GitHub issue.
- Contribution proposal: **Claim a contributor slice** issue.
- Security concern: private process in [Security](SECURITY.md).
- Conduct concern: private process in the [Code of Conduct](CODE_OF_CONDUCT.md).

There is no guaranteed response time, but focused reports with a runnable
reproduction are much easier for contributors to help with.
